# Eventum Workspace OS - Design Direction

## Direction Name

Eventum Workspace OS

Internal shorthand:

- Notion-Warm Workspace Direction

## Product Feeling

The interface should feel like a calm internal workspace for the Eventum team.

It should feel like:

- A private team workspace
- A Notion-like operating system
- A simple work hub
- A page-based productivity system
- A soft dashboard mixed with tasks, routines, notes, meetings, and reports

It should not feel like:

- A heavy SaaS dashboard
- A corporate admin panel
- A dark fintech cockpit
- A CRM
- A futuristic AI app
- A hard execution war room

## Moodboard Interpretation

The moodboard is interpreted as:

- Page-first layouts instead of dashboard-first layouts
- Modular Notion-style blocks
- Soft warm backgrounds
- Clean database-like task rows
- Simple gallery-style project cards
- Meeting and note pages that feel document-like
- Routine sections that feel like practical checklists
- AI as a quiet helper block, not the visual center

## Typography Rules

- Page titles: clear, strong, 28-36px.
- Section titles: quiet, 15-18px.
- Body text: readable 14px.
- Metadata: 11-13px.
- Avoid oversized dashboard numbers except for small metric summaries.
- Avoid excessive uppercase labels.

## Component Rules

Cards:

- Warm white or off-white surfaces.
- Soft 1px border.
- Small to medium radius.
- Minimal shadow.

Badges:

- Small Notion-like tags.
- Pastel backgrounds.
- Low saturation.
- No bright pill overload.

Buttons:

- Primary uses Clay Soil or Deep Mocha.
- Secondary buttons are neutral with subtle borders.
- Ghost buttons are quiet.
- Avoid many filled buttons on one screen.

Tasks:

- Should feel like database rows.
- Use compact metadata.
- Use small status and priority tags.
- Drawer should feel like opening a database item.

Routines:

- Checklist-first.
- Simple done, skipped, and missed states.
- No habit-tracker or gamified streak styling.

Meetings:

- Document-like layout with metadata, notes, decisions, action items, linked tasks, and subtle AI suggestion block.

Notes:

- Most Notion-like area.
- Clean cards and document pages.
- Plain content blocks.

Reports:

- Weekly review page, not analytics dashboard.
- Simple metric blocks, written summary, small lists.

AI:

- Quiet assistant block.
- Structured preview.
- No chatbot-heavy or futuristic treatment.

## Removed Or Reduced

- Dark-heavy cockpit styling
- Purple-led brand treatment
- Neon lime emphasis
- Heavy shadows
- Glowing cards
- Purple gradients
- Dashboard-heavy stat presentation
- Loud badges
- Futuristic AI styling

## Future Design Notes

- Keep the interface page-first and workspace-like.
- Add dark mode later only after the light workspace direction feels stable.
- Build future components from shared tokens rather than one-off colors.
- Preserve operational clarity while keeping the emotional tone calm.
